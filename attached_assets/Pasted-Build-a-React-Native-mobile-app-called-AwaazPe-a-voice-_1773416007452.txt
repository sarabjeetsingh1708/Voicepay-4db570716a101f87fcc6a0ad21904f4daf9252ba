BACKEND API ROUTES:

POST /api/voice/stt — receives base64 audio, sends to Sarvam saarika:v2 model as multipart form-data with fields: file, language_code, model. Returns { transcript: string }
POST /api/voice/parse — parses transcript locally using regex and keyword matching. Returns action, amount, recipient, scheduledDate, confidence. No OpenAI, no external AI API whatsoever.
POST /api/voice/tts — Sarvam text-to-speech using bulbul:v1 model
POST /api/voice/translate — Sarvam translate using mayura:v1 model


VOICE PARSE LOGIC (fully local, no AI API):

Detect action: send, schedule, check_balance, history, unknown
Extract amount from digits or Hindi/English number words:

ek=1, do=2, teen=3, char=4, paanch=5, chhe=6, saat=7, aath=8, nau=9, das=10
bees=20, pachas=50, sau=100, hazaar=1000, lakh=100000


Match recipient by fuzzy name matching against contacts list
Extract schedule date: tomorrow, kal, parso, day names (monday–sunday), date numbers (15th ko)
Confidence score: 0.5 base + 0.2 for send keyword + 0.2 for amount + 0.1 for contact
